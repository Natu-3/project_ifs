1) backwork 실패 내용
----------------------
실행 명령
- .\\gradlew.bat test

실패 증상
- Spring ApplicationContext 로딩 실패
- JPA/Hibernate가 JDBC 메타데이터를 읽지 못함
- 실제 원인 로그에서 DB 사용자 값이 문자열 치환되지 않음

핵심 로그 포인트
- Failed to load ApplicationContext
- Unable to determine Dialect without JDBC metadata
- Access denied for user '${DB_USER}'@'localhost' (using password: YES)

영향받은 설정 변경(직접 원인 가능성 높음)
- backwork/src/main/resources/application.properties
  - spring.datasource.username=${DB_USER}
  - spring.datasource.password=${DB_PASSWORD}

원인 분석
- P1 작업에서 Spring 설정의 fallback (`:root`, `:1234`) 제거
- 테스트 실행 환경에서 `DB_USER`, `DB_PASSWORD` 값을 제공하지 않음
- 결과적으로 `${DB_USER}` / `${DB_PASSWORD}`가 실제 값으로 치환되지 않아 DB 연결 실패

권장 조치
- 테스트 전용 설정에 DB 값 주입 (예: backwork/src/test/resources/application-test.properties)
- 또는 테스트에서 DB 의존 분리(H2/Testcontainers)
- CI/로컬 테스트 실행 스크립트에 env 주입 절차 명시
